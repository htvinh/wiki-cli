Purpose
This procedure defines the process for planning, conducting, reporting, and
| closing | internal | quality | audits | at Acme | Corporation. |     |
| ------- | -------- | ------- | ------ | ------- | ------------ | --- |
Scope
ThisprocedureappliestoallinternalauditsoftheQualityManagementSystem,
| production | processes, |     | and supplier | quality | systems. |     |
| ---------- | ---------- | --- | ------------ | ------- | -------- | --- |
Definitions
InternalAudit: Asystematic,independentexaminationofprocessesandrecords
| to determine | conformity |     | to  | requirements. |     |     |
| ------------ | ---------- | --- | --- | ------------- | --- | --- |
Auditor: A qualified person who performs the audit. Must be independent of
| the area | being | audited. |     |     |     |     |
| -------- | ----- | -------- | --- | --- | --- | --- |
Audit Finding: Evidence of conformity or non-conformity identified during an
audit.
Procedure
| Step 1: | Audit | Planning |     |     |     |     |
| ------- | ----- | -------- | --- | --- | --- | --- |
The QA Manager develops the annual audit schedule by December 15 for the
following year. Each process area must be audited at least once per year. High-
| risk processes |          | are audited  | twice      | per year.         |             |          |
| -------------- | -------- | ------------ | ---------- | ----------------- | ----------- | -------- |
| The audit      | schedule | considers:   |            |                   |             |          |
| • Previous     |          | audit        | results    |                   |             |          |

| • Changes      |          | to processes |            | or personnel      |             |          |
| • Customer     |          | complaints   |            | or quality trends |             |          |
| • Regulatory   |          | requirements |            |                   |             |          |
| Step 2:        | Audit    | Preparation  |            |                   |             |          |
| For each       | audit:   |              |            |                   |             |          |
| 1. Assign      | an       | audit        | team       | leader and        | auditors.   |          |
| 2. Define      | audit    | scope,       | criteria,  | and               | objectives. |          |
| 3. Review      |          | previous     | audit      | reports and       | corrective  | actions. |
4. Prepareauditchecklistsbasedontheapplicablestandardsandprocedures.
| 5. Notify  | the | auditee   | at     | least 2 weeks        | before | the audit date. |
| ---------- | --- | --------- | ------ | -------------------- | ------ | --------------- |
| 6. Provide |     | the audit | agenda | to all participants. |        |                 |
